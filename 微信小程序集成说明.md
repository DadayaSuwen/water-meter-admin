# 微信小程序集成说明

## 概述

本项目已集成微信小程序登录功能，支持通过微信授权快速登录并创建用户账户。

## 功能特性

### 1. 微信登录流程
- 用户在小程序中点击登录
- 小程序调用 `wx.login()` 获取临时登录凭证 `code`
- 前端将 `code` 发送到后端
- 后端调用微信 `code2Session` 接口获取用户的 `openid`
- 系统自动创建或更新用户信息
- 返回JWT Token给前端进行后续认证

### 2. 开发模式支持
- 当微信小程序AppID和AppSecret未配置时，系统自动切换到开发模式
- 开发模式下会生成模拟的openid，便于开发和测试
- 前端可以通过返回的 `isDevelopmentMode` 字段判断当前模式

### 3. 用户信息管理
- 支持在登录时同时设置用户名和手机号
- 支持登录后更新头像、用户名、手机号等信息
- 自动识别用户角色（普通用户/管理员）

## 配置说明

### 1. 环境变量配置

在 `.env` 文件中配置以下变量：

```env
# 微信小程序配置
WX_APP_ID=你的小程序AppID
WX_APP_SECRET=你的小程序AppSecret
```

### 2. 获取微信小程序AppID和AppSecret

1. 登录[微信公众平台](https://mp.weixin.qq.com/)
2. 进入"小程序"模块
3. 选择你的小程序
4. 点击"开发管理" -> "开发设置"
5. 在"开发者ID(AppID)"部分可以看到AppID
6. 点击"重置"获取AppSecret（请妥善保管）

### 3. 服务器域名配置

在微信小程序管理后台配置服务器域名：

- 登录微信公众平台
- 开发管理 -> 开发设置 -> 服务器域名
- 在 `request合法域名` 中添加：`http://127.0.0.1:7001`（开发环境）
- 生产环境需要配置为实际的服务器域名

## API 接口说明

### 1. 微信登录
```
POST /api/auth/login
```

**请求参数：**
```json
{
  "code": "微信登录code",
  "username": "用户名（可选）",
  "phone": "手机号（可选）"
}
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "token": "JWT令牌",
    "user": {
      "id": "用户ID",
      "openid": "微信openid",
      "username": "用户名",
      "phone": "手机号",
      "avatar": "头像URL",
      "role": "用户角色"
    },
    "isDevelopmentMode": false  // 是否为开发模式
  }
}
```

### 2. 检查微信配置状态
```
GET /api/auth/wx-config
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "appId": "已配置",
    "appSecret": "已配置",
    "isComplete": true,
    "message": "微信配置完整，可以使用真实微信登录"
  }
}
```

## 前端集成示例

### 1. 登录页面代码示例

```javascript
// 小程序登录函数
const handleLogin = async () => {
  try {
    // 1. 获取微信登录code
    const loginRes = await wx.login();

    if (loginRes.code) {
      // 2. 调用后端登录接口
      const response = await request('/auth/login', {
        method: 'POST',
        data: {
          code: loginRes.code,
          username: nickname,
          phone: phone
        }
      });

      // 3. 保存token
      wx.setStorageSync('jwt', response.token);

      // 4. 保存用户信息
      wx.setStorageSync('userInfo', response.user);

      console.log('登录成功:', response);
    }
  } catch (error) {
    console.error('登录失败:', error);
    wx.showToast({
      title: '登录失败',
      icon: 'error'
    });
  }
};
```

### 2. 检查配置状态

```javascript
// 检查微信配置
const checkWxConfig = async () => {
  try {
    const response = await request('/auth/wx-config');
    console.log('微信配置状态:', response.data);

    if (response.data.isDevelopmentMode) {
      wx.showModal({
        title: '开发模式',
        content: '当前为开发模式，请配置微信AppID和AppSecret以启用真实登录',
        showCancel: false
      });
    }
  } catch (error) {
    console.error('获取配置状态失败:', error);
  }
};
```

## 错误处理

### 常见错误码说明

1. **40013**: 不合法的AppID
   - 检查AppID是否正确配置

2. **40125**: 不合法的密钥
   - 检查AppSecret是否正确配置

3. **45011**: API调用太频繁
   - 稍后重试

4. **10030**: 无效的openid
   - 可能是code已过期，重新获取

### 错误处理建议

```javascript
try {
  const response = await wxLogin(code);
} catch (error) {
  if (error.message.includes('微信API错误')) {
    // 微信API相关错误
    wx.showModal({
      title: '登录失败',
      content: '微信服务暂时不可用，请稍后重试',
      showCancel: false
    });
  } else {
    // 其他错误
    wx.showToast({
      title: '登录失败',
      icon: 'error'
    });
  }
}
```

## 部署注意事项

### 1. 生产环境配置
- 确保在生产环境中正确配置微信小程序AppID和AppSecret
- 配置正确的服务器域名
- 使用HTTPS协议

### 2. 安全考虑
- AppSecret需要妥善保管，不要提交到代码仓库
- 建议使用环境变量或配置中心管理敏感信息
- 定期更换AppSecret

### 3. 监控和日志
- 监控微信API调用成功率
- 记录登录失败日志，便于排查问题
- 设置告警机制

## 测试方法

### 1. 开发环境测试
- 使用微信开发者工具进行测试
- 确认开发模式下登录功能正常

### 2. 真机测试
- 在微信中打开小程序进行测试
- 验证完整的登录流程

### 3. 配置验证
- 调用 `/api/auth/wx-config` 接口验证配置状态
- 检查控制台日志确认微信API调用情况
