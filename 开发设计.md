# Role
你是一位精通 Node.js 的资深后端架构师，专注于 Midway.js (v3) 框架。你需要为一个微信小程序编写后端服务架构设计。

# Project Context
项目名称：智慧水表小程序后端
核心业务：居民（水表查看、缴费、报修）+ 管理员（审核、管理）。
技术栈要求：
- 框架：Midway.js (v3)
- 语言：TypeScript
- 数据库：**PostgreSQL**
- ORM：**Prisma** (替代 TypeORM，追求更好的类型安全和开发体验)
- 组件：`@midwayjs/prisma`
- 鉴权：JWT (无状态，无 Redis)

# Database Schema (Prisma Schema Design)
请基于 Prisma 的 `schema.prisma` 语法设计数据模型，利用 PostgreSQL 特性：

1. **User (用户)**
   - 字段：id (Int/UUID), openid (String, unique), username, phone, createdAt.
   - **avatar**: String 类型 (映射为 Postgres 的 `Text`)，用于存储 **Base64** 头像。
   - **role**: 使用 Prisma `enum Role { RESIDENT, ADMIN }`。**默认值为 RESIDENT**。
   - 关系：关联 MeterBinding, RepairTicket。

2. **WaterMeter (水表)**
   - 字段：serialNumber (String, unique), location, lastReading (Decimal).
   - **status**: Enum (NORMAL, FAULTY).
   - 关系：关联 MeterBinding, UsageRecord, RepairTicket。

3. **MeterBinding (绑定关系)**
   - 字段：id, status (Enum: PENDING, APPROVED, REJECTED).
   - 关系：User, WaterMeter。
   - 复合唯一约束：确保同一水表同一时间只能被有效绑定。

4. **UsageRecord (用水记录)**
   - 字段：period, amount (Decimal), cost (Decimal), createdAt.
   - 关系：WaterMeter。

5. **RepairTicket (报修工单)**
   - 字段：description (String).
   - **images**: **Json** 类型 (Prisma 原生支持 Postgres JSONB)，存储图片 Base64 或 URL 数组。
   - **status**: Enum (PENDING, PROCESSING, COMPLETED).
   - 关系：User, WaterMeter。

# Functional Requirements (Controller & Service)

## 1. Authentication (JWT & Guard)
- **Login**: 模拟 `code2Session`。如果用户不存在，创建新用户（Role=RESIDENT）。
- **JWT**: 生成包含 `{ userId, role }` 的 Token。
- **Guard**: 
  - `JwtGuard`: 校验 Token。
  - `AdminGuard`: 继承或调用 JwtGuard，并额外检查 `user.role === 'ADMIN'`。

## 2. Feature Implementation (Prisma Client Usage)
- **Resident**:
  - **更新头像**: 接收 Base64 字符串，更新 `User.avatar`。
  - **绑定申请**: `prisma.meterBinding.create` (Status=PENDING).
  - **报修**: `prisma.repairTicket.create` (Images 存入 Json 字段).
- **Admin**:
  - **审核列表**: `prisma.meterBinding.findMany({ where: { status: 'PENDING' }, include: { user: true, waterMeter: true } })`。
  - **审核操作**: 更新状态。如果是 APPROVED，需确保事务一致性。
- **Data**:
  - **平均用水**: 使用 Prisma 的聚合查询 (`_avg`) 计算某水表的平均用水量。
  - **水费计算**: 在 Service 层处理。
